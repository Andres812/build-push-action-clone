Arquitectura del Proyecto

El proyecto sigue una arquitectura basada en automatización DevOps, donde el código fuente pasa por un pipeline CI/CD que valida, construye y prepara el despliegue en contenedores.

CI/CD

CI (Integración Continua)

Cada push ejecuta automáticamente instalación de dependencias, pruebas y build.

CD (Despliegue Continuo)

Después del build, el proyecto puede desplegarse automáticamente usando Docker.

GitHub Actions

Los workflows definen los pasos del pipeline en .github/workflows.
